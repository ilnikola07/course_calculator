****Описание проекта****


Данный репозиторий содержит программное решение для лабораторных работ №4-6. Приложение разработано на языке C# и предназначено для самостоятельного изученияя предложенного графа.


Проект реализован с использованием архитектуры, разделяющей пользовательский интерфейс и бизнес-логику, что обеспечивает модульность и удобство тестирования.


**Основные возможности**


Графический интерфейс (WinForms): удобная форма для загрузки графа и понятные кнопки, исполняющие возможности для исследования графа.


Основные возможности и алгоритмы, реализованные здесь:


Алгоритм BFS (обход в ширину) и вывод порядка посещения вершин


Алгоритм DFS (обход в глубину) и вывести порядок посещения вершин


Определяет достижима ли вершина B из вершины A (с использованием BFS)


Выводит компоненты связности графа


Алгоритм Дейкстры без использования библиотечных функций


Кратчайшее расстояние от заданной вершины до всех остальных


Находит точки сочленения


Минимальное остовное дерево (алгоритм Прима)


Находит кратчайший маршрут выхода из пещеры



**Технологический стек**


Язык программирования: C# версия 12.0


Платформа: .NET Framework 4.8.1


UI Framework: Windows Forms (WinForms)


Архитектура: Class Library (выделение логики в отдельный проект CaveGraph)


Тестирование: xUnit


**Установка и настройка**


Для корректной работы проекта необходимо строго соблюдать структуру каталогов и настройки зависимостей.



1. Получение исходного кода. Склонируйте репозиторий или загрузите архив с исходным кодом по ссылке https://github.com/ilnikola07/alg_lab_4-6_graphs.git.


2. Структура каталогов. Убедитесь, что у вас есть свой файл с графом.


3. Открытие проекта. Запустите Visual Studio (рекомендуется последняя стабильная версия).
Откройте файл решения (.sln), расположенный в папке проекта.


4. Сборка
Выполните команду Build Solution (Собрать решение) для компиляции всех компонентов. При возникновении ошибок сборки проверьте целостность кода и конфигурацию проекта, при надобности просто закомментируйте ошибочные строки внутри странных файлов.


**Использование**


Запустите проект (клавиша F5 или кнопка Start).
В открывшемся окне WinForms выберите номер работы.
Загрузите уже предоставленный граф или воспользуйтесь своим.



**Тестирование**


Проект покрыт модульными тестами с использованием xUnit.
Откройте Test Explorer (Тестовый обозреватель) в Visual Studio (Test -> Windows -> Test Explorer).
Нажмите Run All Tests (Запустить все тесты).
Убедитесь, что все тесты пройдены успешно (отображены зелеными маркерами).
